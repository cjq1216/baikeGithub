# Milestone v1.0: 互动百科 产品化升级 MVP

**Status:** ✅ SHIPPED 2026-06-12

## Overview

将 2017 年 Python 2 时代的 Flask + MySQL 课堂作业(641 宿舍软工大作业)从"能跑通的课堂作业"升级为"可部署演示的产品雏形"——保留核心功能(用户注册登录、词条 CRUD、搜索、评论),通过修复 bug、迁移到 Python 3、现代化前后端、加 Docker 镜像,使另一个开发者仅凭 README 即可拉起并演示全部功能。

**Core Value**: 一个陌生开发者能根据 README 启动 Docker 容器、注册账号、创建词条、发布评论、并以管理员身份看到完整端到端流程。

## Milestone Summary

### Key Decisions (从 PROJECT.md + 各 phase CONTEXT.md)

- 保留 Flask 而非换栈 — 升级目标不是重写(✓ Good)
- 二元角色 (regular + admin),不做 editor/moderator(✓ Good)
- HTMX + Pico.css 而非 React(✓ Good — 单页 + 局部刷新满足需求)
- 词条间 wiki 链接采用 `[[标题]]` 语法(✓ Good — 已在 detail 页 + backlinks 体现)
- 凭据/secret 走环境变量(✓ Good — Phase 2 落地)
- 评论不实装编辑,仅发布 + 作者删除(✓ Good — Scope 守住)
- Docker 不打包 nginx / MySQL,假设外部基础设施(✓ Good — demo 性质匹配)
- 测试仅 smoke, 不写完整测试套件(✓ Good — 3/3 tests 覆盖核心流程)
- 凭据/secret 走 `werkzeug.security` pbkdf2:sha256(✓ Good)
- 全站 Flask-WTF CSRF(✓ Good)
- MySQL PyMySQL shim 兼容 Windows dev(✓ Good — `d4dceb8` 落地)
- `init_db(if_empty=False)` 默认值 + `--if-empty` flag(D-75)幂等容器启动(✓ Good)
- ProxyFix 通过 `FLASK_BEHIND_PROXY=true` 条件启用(D-88,避免 dev 模式 X-Forwarded-For 攻击面)(✓ Good)
- wait-for-mysql 用 stdlib socket 重试 30s,零外部依赖(D-76)(✓ Good)
- TEST-05 验收清单嵌入 README 单文件(D-79),不拆 ACCEPTANCE.md(✓ Good)
- SQLite in-memory 测 fixture 不引入 factory(CD-26,Phase 5 demo 性质)(✓ Good)
