# Plan 04-01: 前端栈替换 (jQuery 移除 + Pico.css + HTMX + Quill)

<objective>
把前端栈从 jQuery 1.11 + Bootstrap 3 + wangEditor 2.x 升级到 Pico.css 2.0.6 (CDN+SRI) + HTMX 1.9.10 (CDN+SRI) + Quill 2.x (本地 vendor)。具体:删除全部 jQuery / Bootstrap 3 / wangEditor / mycss 旧静态资产;新增 `base.html` 共享布局;add.html 和 modify.html 用 Quill 2.x 富文本编辑器(8 按钮精简工具栏);`/api/add` 和 `/api/modify` 路由用 bleach 库白名单过滤 HTML 防 XSS;从 add.html / modify.html 抽 `_comment.html` 子模板供 Plan 4.3 评论 HTMX swap 复用。

本 plan 是 Wave 1(单 plan),后续 Plan 4.2/4.3 依赖本 plan 的 `base.html` 和已删除的旧静态资源。
</objective>

<must_haves>
- [ ] SC-1: 无 jQuery / Bootstrap 3 CSS/JS 加载,旧 `jquery-1.11.3.min.js` / `bootstrap.min.js` / `bootstrap.min.css` / `bootstrap-theme.min.css` / `wangEditor.min.js` / `wangEditor.min.css` 资产全部从磁盘删除
- [ ] SC-2: Pico.css 2.0.6 + HTMX 1.9.10 通过 jsdelivr CDN + SRI 哈希 + `crossorigin="anonymous"` 加载,`base.html` 是 7 子模板共享布局
- [ ] SC-3: add.html 和 modify.html 用 Quill 2.x(本地 vendor,8 按钮精简工具栏:粗体/斜体/下划线 + 有序/无序列表 + H1-H3 + 引用 + 链接 + 清除格式)替换 wangEditor,form 提交前 JS 把 Quill HTML 同步到 hidden `<textarea name="content">`
- [ ] INFRA-12(防 XSS):`/api/add` 和 `/api/modify` 路由在保存 DB 前用 `bleach.clean(content, tags=ALLOWED_TAGS, attributes={'a': ['href']}, protocols=['http','https'])` 过滤;白名单标签含 `p / b / i / u / strong / em / a / ul / ol / li / h1 / h2 / h3 / br / blockquote / pre / code`
- [ ] 7 子模板(本 plan 重写 add/modify + 创建 _comment;Plan 4.2 重写其余 5 个)开始 `{% extends 'base.html' %}` 而非各自独立的 `<!DOCTYPE html>` 全栈

   - Mitigation: **D-44 bleach 白名单**(`/api/add` 和 `/api/modify` 调 `bleach.clean(content, tags=ALLOWED_TAGS, attributes={'a': ['href']}, protocols=['http','https'])`)。`ALLOWED_TAGS` 不含 `script` / `iframe` / `object` / `embed` / `style` / `form`;`javascript:` 协议被 protocols 白名单拒(被 strip);`onclick` / `onerror` 等事件属性不在 attributes 白名单(被 strip)。`{{ fullcon.content | safe }}` 在 detail.html 仍存在(Plan 4.3 实施),但 DB 存的内容已脱 XSS。
   - Test: `python -c "import bleach; print(bleach.clean('<script>alert(1)</script><p>safe</p>', tags=['p','b','i','u','strong','em','a','ul','ol','li','h1','h2','h3','br','blockquote','pre','code'], strip=True))"` 输出 `'<p>safe</p>'`。

2. **T2: CDN compromise (Pico.css / HTMX SRI failure)**
   - Vector: 攻击者攻陷 jsdelivr CDN 或中间人替换响应内容,注入恶意 JS / CSS,在用户浏览器执行。
   - Mitigation: **D-53 SRI 哈希**(`integrity="sha384-..."` + `crossorigin="anonymous"`)在 base.html `<link>` 和 `<script>` 标签上。浏览器加载资源时先校验 hash,不符则拒绝加载。SRI 哈希**必须**用 `curl -sI <CDN URL> | grep -i 'x-sri'` 从 jsdelivr 实际响应头取真实 sha384 哈希(Pico.css 与 HTMX 各一);**不**留 `sha384-PLACEHOLDER` 或空字符串(空 hash 等于不验证,placeholder 形同虚设)。**fetch 失败则任务不 done**,executor 应记 blocker 交回 orchestrator(不接受"占位后期替换"作为 done 状态)。
   - Test: base.html 含 `integrity="sha384-` 前缀(Pico + HTMX 各一),且**不**等于字面量 `sha384-PLACEHOLDER`(任务验收硬约束)。

3. **T3: CSRF on new HTMX forms**
   - Vector: 攻击者构造跨站请求,在用户已登录情况下提交 `/api/add` 创建垃圾词条。
   - Mitigation: **D-09 携带:Flask-WTF CSRF 全局保护**。add.html 和 modify.html `<input type="hidden" name="csrf_token" value="{{ csrf_token() }}">` 不可省;`/api/add` / `/api/modify` 路由继承 Phase 2 `CSRFProtect(app)`,无 token POST 返 400 → 走 D-11 `handle_csrf_error` flash 重定向。
   - Test: 模板含 `name="csrf_token"`;`grep -F 'csrf_token()' app/templates/add.html` / modify.html 退出 0。

4. **T4: 主题切换按钮可访问性(非安全,但是 a11y 兜底)**
   - Vector: 屏幕阅读器无法识别图标按钮。
   - Mitigation: `<button id="theme-toggle" aria-label="切换主题">` aria-label 标注(CD-15 + a11y 兜底)。

5. **T5: 旧资产引用残留(攻击面没拆干净)**
   - Vector: Plan 4.1 删了 `app/static/javascripts/jquery-1.11.3.min.js` 但某个未重写模板还有 `<script src="...jquery...">`,404 不致命但暴露旧版本号。
   - Mitigation: **D-55 完整性** + **verification step 1** (`find app/static` grep 全无遗留资产) + **verification step 3** (`grep -r 'wangEditor\|bootstrap\|jquery' app/templates/` 输出为空)。

Risk: T1 + T3 必须严格执行;T2 / T5 是依赖具体执行质量。
