<objective>
Eliminate three latent runtime bugs and one code smell in the application code itself: (1) `/api/modify` is missing a `return` and uses `db.session.merge()` as a side-effect INSERT instead of UPDATE; (2) `/user/detail` passes a `BaseQuery` to the template instead of a list; (3) all three SQLAlchemy models have a `__tablenanme__` typo; (4) `app/api/model.py` constructs a duplicate `Flask(__name__)` instance, dead code that creates confusion about app ownership. All fixes are surgical — no new features, no refactoring of unrelated code.

Purpose: Phase 1's second half (INFRA-03, INFRA-04). These bugs are silent on Python 2 / happy-path code; they cause 500s and `NoneType` errors on Python 3 + Flask 3.x. With the dependency migration in Plan 1.1 already done, this plan makes the runtime actually work end-to-end.

Output:
- `app/api/model.py` with `db = SQLAlchemy()` (no app arg) and three corrected `__tablename__` declarations.
- `app/api/__init__.py` `/api/modify` handler that looks up the existing lemma by title, mutates it, commits, and redirects.
- `app/route/user.py` `/user/detail` handler that materializes a list and guards the empty case.
- A smoke verification confirming the end-to-end flow works.
</objective>
